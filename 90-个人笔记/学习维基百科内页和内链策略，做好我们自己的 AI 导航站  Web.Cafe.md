大家周一下午好，预告一下，一会儿15点，久违的#哥飞小课堂 又开课了，这次跟大家聊聊维基百科的内页和内链设计。 我们做AI导航站，就特别适合学习借鉴这种模式。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/c4a1793b4fcb4c2ab60c3e525b6bd256.jpg)

聊维基百科内链之前，先跟大家聊聊品牌命名。

我们要如何给自己的产品命名，如果做一个有自己品牌名称的产品，而不是简单的用关键词做域名。

最典型的例子是 Cursor ，本来这个词是鼠标的光标的意思，但现在，这个词已经被一家公司做的产品给抢走了，变成了产品的品牌词。

这件事情，很有意思。

大家可以想一下，这个词在互联网上已经出现了很多年了，突然，某一天，一家公司宣布说，我的产品名称就用这个词了，于是过了一段时间，谷歌就让这家公司的网站排到了第一名。

这，是不是很有意思？

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/f2a433028f814b56a20539532b85ac44.jpg)

再看一个例子，在谷歌搜索 Mapify ，现在排第一名的网站是 mapify.so 。 而在去年的5月，这个词搜索第一名是 mapify.ai 。 从这个案例可以看到，别人已经在用的品牌名称，居然也可以抢走。 甚至以 .so 这个非主流域名，抢走了 .ai 域名的排名。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/11a987efa4c6441a84102ea8454deffc.jpg)

再看第三个例子，谷歌搜索Flux，前三个结果，第一个是很有名的屏幕色温调节软件，第二名是一个用AI去设计PCB电路板的产品，第三名是Flux AI生图网站。 也就是说，谷歌是允许多个产品公用一个品牌名称的。

大家可以思考一下，为什么谷歌要这么做？

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/b6a43f3e260743da99dffa3145f4ba7b.jpg)

好了，15点到了，今天的#哥飞小课堂 正式开始，在 Cursor 的搜索结果里，第二名就是维基百科的页面。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/08f393efafd44567ad52bbf8429b0194.jpg)

我们打开页面，就可以看到维基百科的经典页面之词条详情页。 维基百科网站的绝大多数的流量，都靠这个词条详情页来承接。 不管是站外搜索（如谷歌、百度、Bing）还是站内搜索，用户想找的其实就是这个词条详情页。

换句话说，这个页面，既能够满足用户需求，又能够拿到搜索排名。

那么，问题就来了，为什么这个页面设计，可以拿到排名呢？

从 Headings 结构，可以得到答案，围绕着当前词条，这个页面详细的从各个角度来阐述解释跟这个词条相关的内容。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/bfc56d474dfc47a9ae02a6fdea7ddbba.jpg)

也就是说，用户只需要看这个页面，就能够大概了解到Cursor这个产品的所有信息。 其他词条页面同理，只要是跟词条相关的内容，都会被编辑进去，形成一个内容特别丰富又特别相关的页面。

维基百科还有一个特色设计，就是大量的内链。 任意一个词条页面，提到了别的词条，都会链接过去。 最终形成了语义话的复杂页面网络，你中有我，我中有你，互相连接，互相解释。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/88aabab80bf34d2cbb04d456a41786ce.jpg)

除了词条与词条在正文中互相链接之外，维基百科每个词条内页，还有 See Also 设计，列出来一些跟当前词条相关的词条页面。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/29ed1823b4474c01a86b464b5f5ba68c.jpg)

在词条内容底部，还有当前词条的所属分类，一个词条可以处于多个分类下。 这也是很有意思的设计，因为一个词条，就是有可能在不同的属性下，在不同的分类里。 举个电影的例子，逃学威龙，可以在以下分类里： 1991电影 1991香港电影 陈嘉上电影 周星驰电影 粤语电影 ……

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/e404cddee21a4691aa93ff607516a96f.jpg)

这个设计，不仅仅可以迁移到AI导航站里，还可以迁移到AI工具站里。 时不时有朋友问，我同一个工具，是否可以做多个页面？ 其实是可以的。 举例，你做了一个在线图片编辑器，那么可以建立以下页面： Online Image Editor Online Photo Editor Online Picture Editor 只要是有人搜索的关键词，你都可以为这个关键词做一个页面，围绕着不同的关键词写不同的文案，但实际使用同一个功能来满足需求。

回到刚才 Cursor 的分类，我们打开 IDE 这个分类页面，可以看到，在大分类页面，列出来了小分类列表，然后下方列出来了IDE分类下所有的词条列表。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/87d3ec71099c4217b05fb36ee3aa6143.jpg)

通过词条、分类等页面之间的合理内链，维基百科这么庞大的网站，流量就流动起来了。 同时，也让搜索引擎的爬虫，可以更好的爬到所有的页面。

分类页面，满足的是了列出信息的需求。 举例，当我想要了解，2020年有哪些公司合并事件发生，我去谷歌搜索 2020 merger companies ，排到第二第三名的，就是维基百科的分类页面。 在这些页面里，就列出来了，我要要知道的公司列表。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/4a936210c5e14ec6a48a412b53cef18e.jpg)

来回答一下，刚才小课堂之前的几个问题：

### 1\. 为什么mapify.so打败了mapify.ai，拿到了第一名，并且拿到了Sitelinks？

答：虽然 mapify.ai 先上线，但是 mapify.so 的品宣和外链太强了。 当全网在讨论 Mapify 时，提到的域名都是 mapify.so ，于是谷歌就认为，Mapify 这个词，应该是 mapify.so 。

![图片](%E5%AD%A6%E4%B9%A0%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E5%86%85%E9%A1%B5%E5%92%8C%E5%86%85%E9%93%BE%E7%AD%96%E7%95%A5%EF%BC%8C%E5%81%9A%E5%A5%BD%E6%88%91%E4%BB%AC%E8%87%AA%E5%B7%B1%E7%9A%84%20AI%20%E5%AF%BC%E8%88%AA%E7%AB%99%20%20Web.Cafe/2ea76ab2a3a34da8975fc79911b52353.jpg)

Cursor 同理，全网在讨论 Cursor 时，绝大多数内容指向的都是 Cursor IDE ，而不是别的，于是Cursor 就跟cursor.com绑定在一起了。

### 2\. 为什么谷歌允许多个产品用相同的名字？

因为谷歌无法控制大家如何命名自己的产品，既然管不了，索性就不管。

那么，谷歌要解决的是，如何根据互联网上的内容，判断相同名称的网站，谁要排到前面，谁要排到后面。

只要这样排序后，能够满足谷歌的搜索用户需求，那么就可以了。

谷歌没必要当裁判，说A域名先用A名称的，后面其他网站都不能使用了。

互联网有自己的新陈代谢，曾经风光的A网站，也许后面就倒闭没落了，此时就应该允许另一个也叫A的网站出现，甚至就使用A曾经用过的域名都可以。
